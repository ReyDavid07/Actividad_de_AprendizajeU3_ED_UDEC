# Unidad 3 - Ejercicio 4 - Restaurante

Sistema de gestión de pedidos de restaurante usando colecciones del SDK de Java: List, Queue, Deque, Map y Stream.

## Caso asignado
- Ejercicio: 4
- Caso: Restaurante
- Entidad principal: Pedido
- Identificador: Número de pedido

## Colecciones usadas
- List: registro general de pedidos.
- Queue: pedidos pendientes por procesar, aplicando FIFO.
- Deque: historial tipo pila de pedidos procesados, aplicando LIFO.
- Map: búsqueda rápida por número de pedido.
- Stream: búsquedas, filtros, ordenamientos, transformaciones, conteos, agrupamientos y estadísticas.
